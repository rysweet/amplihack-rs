# Code Atlas

Living architecture documentation for the amplihack-rs repository, generated by
the [code-atlas recipe](../../amplifier-bundle/recipes/code-atlas.yaml).

## Layers

| # | Layer | Description |
|---|-------|-------------|
| 1 | [Repo Surface](repo-surface/README.md) | Source files, project structure, build systems |
| 2 | [AST / LSP Bindings](ast-lsp-bindings/README.md) | Cross-file symbol references, dead code |
| 3 | [Compile Dependencies](compile-deps/README.md) | Package imports, dependency trees |
| 4 | [Runtime Topology](runtime-topology/README.md) | Services, processes, connections |
| 5 | [API Contracts](api-contracts/README.md) | CLI commands, hook protocol, recipe steps |
| 6 | [Data Flow](data-flow/README.md) | DTO-to-storage chains, transformation steps |
| 7 | [Service Components](service-components/README.md) | Per-crate module structure |
| 8 | [User Journeys](user-journeys/README.md) | End-to-end sequence diagrams |

## Diagram formats

Each layer includes:

- `.dot` source (Graphviz) and rendered `-dot.svg`
- `.mmd` source (Mermaid) for rendering with `mmdc`

## OpenCypher

Standalone OpenCypher files for loading the atlas into any graph database are in
[cypher/](cypher/).

## Regenerating

```bash
amplihack recipe run amplifier-bundle/recipes/code-atlas.yaml
```

Or via the GitHub Actions workflow (`.github/workflows/atlas.yml`) which runs on
every push to `main` and uploads the atlas as an artifact.
