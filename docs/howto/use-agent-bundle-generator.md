# Use the Agent Bundle Generator

> **Status: Mixed.** The agent generation step (`amplihack new`) is
> shipped. The bundle/package/distribute steps are stubs in
> `crates/amplihack-cli/src/cli_extensions.rs` and the
> `amplihack agent-generator bundle …` CLI proposed on this page is **not
> yet wired**. Use the programmatic stubs at your own risk; their output
> is intentionally minimal until the implementation lands.

This how-to threads the existing generator with the planned bundle layer.

## What works today

```sh
# Generate a goal agent on disk (this works)
echo "An agent that summarizes pull requests" > prompt.md
amplihack new --file prompt.md --output ./goal_agents
```

This produces a directory under `./goal_agents/` containing the agent
manifest, skills, and templates, generated by the four-stage pipeline
described in [Goal Agent Generator](../concepts/agent-generator.md).

## What the bundle layer adds (stubs today)

The library functions in `cli_extensions.rs` define the intended shape:

```rust
use amplihack_cli::cli_extensions::{generate_bundle, package_bundle, distribute_bundle};

// 1. Wrap the generated agent into a bundle directory
let bundle_dir = generate_bundle(
    "An agent that summarizes pull requests",
    Path::new("./bundles/pr-summarizer"),
    /* validate */ true,
    /* test */ true,
)?;

// 2. Package it (currently writes a placeholder file)
let archive = package_bundle(
    &bundle_dir,
    "tar.gz",
    Path::new("./dist/pr-summarizer.tar.gz"),
)?;

// 3. Distribute (currently no-op stub)
distribute_bundle(
    &archive,
    /* github */ true,
    /* pypi */ false,
    /* local */ false,
    /* release */ false,
)?;
```

The validate / test / distribute flags are accepted by the signature but
**do not perform real work yet**. Treat the call sites as TODO anchors.

## Planned CLI

When implementation lands, the proposed surface is:

```text
# Planned — not implemented yet
amplihack agent-generator bundle   --prompt-file prompt.md --out ./bundles/foo
amplihack agent-generator package  --bundle ./bundles/foo --format tar.gz
amplihack agent-generator publish  --package ./dist/foo.tar.gz --github
```

Until those subcommands exist, drive the pipeline through `amplihack new`
plus direct calls to `cli_extensions`.

## Verify

For the parts that work today:

```sh
ls ./goal_agents/<generated-name>/   # bundle directory
cat ./goal_agents/<generated-name>/manifest.json
```

There is intentionally no "verify the package" step in this how-to: the
packager is a stub and producing a fake checksum here would violate
zero-BS.
